# Segurança — FreteHub

## Autenticação

### JWT (SimpleJWT)
- **Access Token Lifetime**: 60 minutos
- **Refresh Token Lifetime**: 7 dias
- **Rotate Refresh Tokens**: sim (rotação automática)
- **Blacklist After Rotation**: sim (invalidar tokens usados)
- **Update Last Login**: sim
- **Algoritmo**: HS256
- **Header Type**: Bearer

### Modelo de Usuário Customizado
- Login por `email` (não username)
- Campos: name, email, phone, document_type, document_number
- Tipos de usuário com permissões diferenciadas
- `is_verified` para confirmação de conta
- `is_active` para habilitação/desabilitação

## Autorização e Permissões

### Tipos de Usuário e Acessos
| Tipo | Acesso |
|------|--------|
| ADMIN | Total (Admin) |
| SUPPORT | Logs de auditoria, suporte |
| FINANCE | Operações financeiras |
| LOGISTICS | Operações logísticas |
| PF/PJ | Apenas seus próprios recursos |
| DRIVER | Recursos de motorista |
| CARRIER | Recursos de transportadora |

### Endpoints Protegidos
Todos os endpoints exceto:
- `POST /api/auth/register/`
- `POST /api/auth/login/`
- `POST /api/auth/refresh/`

## Auditoria

### AuditMiddleware
Middleware automático que registra:
- Usuário responsável
- Ação realizada
- Tipo de entidade afetada
- ID da entidade
- Endereço IP
- User Agent
- Metadados adicionais (JSON)

### Modelo AuditLog
Registra todas as operações importantes do sistema para:
- Conformidade regulatória
- Investigação de incidentes
- Rastreamento de mudanças
- Responsabilização de ações

## Segurança de Rede

### Configurações Docker
- PostgreSQL: porta 5432 (interna)
- Redis: porta 6379 (interna)
- Django: porta 8000
- Daphne/ASGI: porta 8001
- pgAdmin: porta 5050
- Mailhog: porta 8025/1025

### Variáveis de Ambiente Sensíveis
- `SECRET_KEY` — Chave secreta Django
- `JWT_SECRET` — Chave de assinatura JWT
- `DATABASE_URL` — String de conexão PostgreSQL
- `REDIS_URL` — String de conexão Redis

## Boas Práticas Implementadas

1. **Campos de Senha** — `set_password()` via Django (hash bcrypt)
2. **Tokens com Blacklist** — Refresh tokens invalidados após uso
3. **UUID como PK** — Não expõe IDs sequenciais
4. **Soft Deletes** — Modelo `is_active` ao invés de deleção
5. **Campos de Auditoria** — created_at, updated_at em todos os modelos
6. **CORS Configuration** — django-corsheaders configurado
7. **CSRF Protection** — Middleware Django CSRF ativo
8. **JSONRenderer Padrão** — Respostas padronizadas em JSON
