# Implementação — FreteHub

## Stack Tecnológica

| Camada | Tecnologia |
|--------|------------|
| Linguagem | Python 3.11 |
| Framework Web | Django 4.x + Django REST Framework |
| Banco de Dados | PostgreSQL 15 + PostGIS |
| Cache/Mensageria | Redis 7 |
| Filas de Tarefas | Celery + Celery Beat |
| Servidor ASGI | Daphne |
| WebSocket | Django Channels (InMemory) |
| Autenticação | SimpleJWT (JWT) |
| Containerização | Docker + Docker Compose |

## Arquitetura de Aplicações (apps/)

O projeto utiliza arquitetura modular Django com os seguintes módulos:

### Módulos Fundamentais
- **core/** — Classe base `UUIDModel` (UUID pk, created_at, updated_at), renderers padrão
- **accounts/** — Modelo `User` customizado (login por e-mail), `UserConsent`
- **companies/** — `CompanyProfile` (PJ genérico), `CarrierCompany` (transportadoras)
- **audit/** — `AuditLog` + `AuditMiddleware` para logging automático de ações

### Módulos Operacionais
- **drivers/** — `DriverProfile` (CNH, categoria, avaliação, geolocalização, PIX)
- **vehicles/** — `Vehicle` (tipo, carroceria, capacidade peso/volume, RENAVAM, seguro)
- **cargo/** — `CargoType` (categorias: Equipamentos, Grãos Secos), `CargoRule` (requisitos logísticos)
- **freight/** — `FreightOrder` (ordem de frete completa), `CargoItem` (itens individuais)

### Módulos de Inteligência e Automação
- **pricing/** — `PriceEstimate` (tarifa base, distância, tempo, ajudante, seguro, pedágio)
- **matching/** — `MatchCandidate` (score compatibilidade, proximidade, avaliação)
- **routing/** — `Route` (polyline, restrições operacionais)
- **tracking/** — `TrackingEvent` (eventos geolocalizados com foto, assinatura)
- **eta/** — `ETARecord` (previsão de chegada dinâmica)
- **ai_assistant/** — `AIConversation`, `AIMessage`, `AIAction`, `KnowledgeDocument`, `KnowledgeEmbedding`

### Módulos de Suporte e Negócio
- **payments/** — `Payment` (métodos: PIX, cartão, boleto, faturamento)
- **documents/** — `Document` (CNH, CRLV, Nota Fiscal, Pesagem)
- **notifications/** — `Notification` (tipos por evento de frete)
- **support/** — `SupportTicket` (categorias, prioridades, status)
- **commercial/** — `InvestorLead` (gestão de leads comerciais)

## Configurações (config/)

- **settings/base.py** — Configurações centrais (INSTALLED_APPS, MIDDLEWARE, DRF, JWT, Celery)
- **settings/development/** — Overrides para desenvolvimento local
- **settings/production/** — Overrides para produção
- **urls.py** — Roteamento raiz de URLs
- **asgi.py** — Protocol Type Router para Channels/ASGI
- **wsgi.py** — Configuração WSGI síncrona
- **celery.py** — Instanciação do Celery

## Modelos de Dados — Relacionamentos Principais

```
User (accounts)
├── CompanyProfile (companies) — OneToOne
├── CarrierCompany (companies) — OneToOne
├── DriverProfile (drivers) — OneToOne
│   └── Vehicle (vehicles) — ForeignKey (carrier)
├── AIConversation (ai_assistant) — ForeignKey
├── Notification (notifications) — ForeignKey
├── SupportTicket (support) — ForeignKey
└── Document (documents) — ForeignKey

FreightOrder (freight)
├── CargoItem (freight) — ForeignKey
├── Route (routing) — OneToOne
├── TrackingEvent (tracking) — ForeignKey
├── ETARecord (eta) — ForeignKey
├── Payment (payments) — ForeignKey
├── MatchCandidate (matching) — ForeignKey
└── SupportTicket (support) — ForeignKey

Vehicle (vehicles)
├── DriverProfile (drivers) — ForeignKey
└── TrackingEvent (tracking) — ForeignKey
```

## Fluxo Principal de um Frete

1. **Cliente** solicita frete via API (`freight/`)
2. **Pricing** calcula estimativa de preço (`pricing/`)
3. **Matching** encontra candidatos compatíveis (`matching/`)
4. **Motorista** aceita e inicia rota (`freight/status`, `routing/`)
5. **Tracking** registra eventos de localização (`tracking/`)
6. **ETA** atualiza previsão de chegada (`eta/`)
7. **Pagamento** é processado (`payments/`)
8. **Notificação** enviada ao cliente (`notifications/`)
